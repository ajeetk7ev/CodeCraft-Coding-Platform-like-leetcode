### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the bug in detail and provide steps to reproduce.
- Include information about your environment (OS, Browser, Node version).

### Suggesting Enhancements
- Check if the feature has already been suggested.
- Open an issue with the "enhancement" label.
- Explain why this feature would be useful to most users.

### Pull Requests
1. **Fork the Repo**: Create your own copy of the project.
2. **Setup**: Follow the [README.md](./README.md) instructions to get the project running.
3. **Branch**: Create a new branch for your feature or fix (`git checkout -b feature/amazing-feature`).
4. **Code**: Write clear, documented, and tested code.
5. **Commit**: Use descriptive commit messages.
6. **Push**: Push your changes to your fork.
7. **Submit**: Open a Pull Request against the `main` branch.

## ğŸ¨ Style Guidelines

### Git Commit Messages
- Use the present tense ("Add feature" not "Added feature").
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
- Limit the first line to 72 characters or less.

### Code Style
- We use **ESLint** and **Prettier** for code formatting.
- Ensure your code passes all linting checks before submitting.
- Use meaningful variable and function names.
